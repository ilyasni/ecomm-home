# Project Knowledge Base

## Назначение

Единая база проектных знаний для фиксации решений по задачам, архитектурных компромиссов и практик внедрения.

## Структура

- `tasks/` — отчёты по выполненным задачам и внедрениям.
- `decisions/` — архитектурные решения и их эволюция.

## Правила ведения

- Для каждой значимой задачи создавай запись в `tasks/` на основе шаблона.
- Для долгоживущих архитектурных решений добавляй отдельный документ в `decisions/`.
- Не дублируй большие куски кода, ссылайся на файлы/коммиты/документы.
- Обязательно фиксируй проверки и rollback-план.

## Быстрый старт

1. Скопируй `tasks/TEMPLATE.md` в новый файл с именем формата `YYYY-MM-DD-short-topic.md`.
2. Заполни секции: Context, Solution, Checks, Impact/Rollback.
3. При необходимости добавь ссылку на связанное решение в `decisions/`.
